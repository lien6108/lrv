## 1. 系統簡介

**實價登錄查詢系統**是一套台灣不動產實際交易價格查詢平台，資料來源為內政部地政司「不動產交易實價查詢服務網」，涵蓋：

| 資料類型 | 說明 | 代碼 |
|---------|------|------|
| 買賣成交 | 一般房屋買賣成交紀錄 | A |
| 預售屋 | 預售屋交易紀錄 | B |

系統提供：
- 多條件篩選查詢
- 自訂顯示欄位
- 排序與分頁
- 每月定期自動下載最新資料
- 資料更新完成後發送 Email 通知

## 2. 啟動系統

### 安裝依賴

```bash
pip install -r requirements.txt
```

### 啟動伺服器

```bash
python main.py
```

伺服器啟動後，開啟瀏覽器前往：

```
http://localhost:8000
```

> **首次啟動**：若專案目錄中存在 `row data.xlsx`，系統會自動匯入初始資料，並建立 `.initial_import_done` 標記檔，避免重複匯入。

## 6. 篩選條件面板

### 折疊 / 展開

點擊「**篩選條件 ▾**」列可折疊或展開整個篩選面板，方便小螢幕使用。

當有任何篩選條件啟用時，標題旁會顯示 **數字徽章**（例如 `3`），提示目前生效的篩選數量。

---

### 第一列篩選條件

| 元件 | 類型 | 說明 |
|------|------|------|
| **鄉鎮市區** | 下拉選單 | 依行政區篩選，選項由資料庫動態產生 |
| **建物型態** | 下拉選單 | 依建物類型篩選，選項由資料庫動態產生 |
| **社區案名** | 文字輸入 | 模糊搜尋（部分符合即可），例如輸入「信義」會找到所有案名包含「信義」的紀錄 |
| **民國年** | 數字輸入 | 輸入民國年份，例如 `112`（代表 2023 年） |
| **月份** | 數字輸入 | 輸入 1～12 的月份數字 |

---

### 第二列篩選條件

| 元件 | 類型 | 說明 | 單位 |
|------|------|------|------|
| **總價最小** | 數字輸入 | 交易總價下限 | 萬元 |
| **總價最大** | 數字輸入 | 交易總價上限 | 萬元 |
| **單價最小** | 數字輸入 | 每平方公尺單價下限 | 元/m² |
| **單價最大** | 數字輸入 | 每平方公尺單價上限 | 元/m² |
| **房數** | 數字輸入 | 精確符合的房間數，例如 `3` 代表 3 房 | 間 |
| **衛數** | 數字輸入 | 精確符合的衛浴數，例如 `2` 代表 2 衛 | 間 |

---

### 查詢與清除按鈕

| 按鈕 | 外觀 | 功能 |
|------|------|------|
| **查詢** | 藍色實心按鈕 | 套用所有篩選條件並執行查詢，頁碼重置為第 1 頁 |
| **清除** | 灰色邊框按鈕 | 清除所有篩選條件，重新載入全部資料 |

> **快捷鍵**：在任何篩選輸入框中按下 `Enter` 鍵，效果等同點擊「查詢」按鈕。

## 8. 欄位設定（Column Studio）

### 開啟面板

點擊統計列右側的「**欄位設定**」按鈕，側邊面板從右側滑入。同時頁面出現半透明遮罩，點擊遮罩也可關閉面板。

### 欄位分組

面板內欄位依以下 8 個分類排列，每個分類可個別勾選：

| 分組 | 包含欄位 |
|------|---------|
| 基本資訊 | 鄉鎮市區、交易年月、來源… |
| 地址 | 區段、門牌… |
| 建物 | 建物型態、主要用途、建材… |
| 格局 | 房廳衛數、隔間… |
| 面積 | 各種面積欄位 |
| 價格 | 總價、單價欄位 |
| 車位 | 車位相關欄位 |
| 其他 | 備註、移轉編號… |

### 欄位勾選

- ✅ 勾選（深色標籤）：該欄位顯示於結果表格中
- ☐ 取消勾選（灰色標籤）：該欄位隱藏
- 勾選或取消後，表格**立即更新**，無需點擊確認。

### 面板按鈕

| 按鈕 | 功能 |
|------|------|
| **恢復預設** | 恢復預設的 9 個顯示欄位（★ 標示欄位） |
| **完成** | 關閉面板，儲存目前設定 |
| **✕（關閉圖示）** | 同上，關閉面板 |

> **設定持久化**：欄位顯示設定儲存於瀏覽器 `localStorage`，重新整理或關閉瀏覽器後下次開啟仍維持相同設定。

## 9. 分頁控制

分頁列位於查詢結果表格的下方。

| 元件 | 說明 |
|------|------|
| **← 上一頁** | 切換到前一頁；在第 1 頁時為灰色不可點擊 |
| **第 X 頁，共 Y 頁** | 顯示目前頁碼與總頁數 |
| **下一頁 →** | 切換到下一頁；在最後一頁時為灰色不可點擊 |

切換頁面時，表格畫面自動捲回頂部。

## 10. 自動資料同步

系統內建排程，**每月 1 日、11 日、21 日 00:00（台北時間）**自動執行資料下載與匯入。

### 資料流程

```
排程觸發
  ↓
向政府資料庫下載最新 CSV
  ↓
解析欄位、換算單位（m² → 坪）
  ↓
以「移轉編號」為唯一索引，避免重複匯入
  ↓
批次寫入資料庫
  ↓
發送 Email 通知
```

### 手動觸發匯入

若需立即更新資料，可呼叫以下 API（需自行透過工具或程式呼叫）：

```
POST /api/import/trigger
```

## 11. Email 通知

每次資料更新完成後，系統會發送 HTML 格式通知信到設定的收件信箱。

### 信件格式

| 欄位 | 內容 |
|------|------|
| 主旨 | `【實價登錄】資料更新完成，本次新增 XXXX 筆` |
| 內文 | HTML 表格，含各來源（A/B）狀態、新增筆數、觸發方式、完成時間 |

### 設定收件信箱

編輯根目錄的 `.env` 檔，設定以下環境變數：

```env
GMAIL_USER=你的Gmail帳號@gmail.com
GMAIL_APP_PASSWORD=你的應用程式密碼
NOTIFY_EMAIL=收件人信箱@example.com
```

> Gmail 需啟用「**應用程式密碼**」（App Password），而非帳號原始密碼。

## 12. API 端點總覽

系統提供 RESTful API，可供其他程式或工具整合使用。

### GET /api/transactions — 查詢交易紀錄

| 參數 | 類型 | 說明 |
|------|------|------|
| `district` | string | 鄉鎮市區（精確符合） |
| `year` | int | 民國年（例如 `112`） |
| `month` | int | 月份（1–12） |
| `community` | string | 社區案名（模糊搜尋） |
| `building_type` | string | 建物型態（精確符合） |
| `min_price` | int | 總價下限（元） |
| `max_price` | int | 總價上限（元） |
| `min_unit_price` | float | 單價下限（元/m²） |
| `max_unit_price` | float | 單價上限（元/m²） |
| `bedrooms` | int | 房數（精確符合） |
| `bathrooms` | int | 衛數（精確符合） |
| `sort_by` | string | `total_price` / `unit_price` / `year_month` |
| `sort_dir` | string | `asc` / `desc`（預設 `desc`） |
| `page` | int | 頁碼（預設 `1`） |
| `limit` | int | 每頁筆數（預設 `20`，最大 `100`） |

### GET /api/stats — 儀表板統計

回傳總筆數、來源分佈（A/B）、前 8 建物型態分佈。

### GET /api/options — 篩選選項

回傳所有可用的鄉鎮市區、建物型態選項清單，以及最後匯入時間。

### POST /api/import/trigger — 手動觸發匯入

在背景啟動資料下載與匯入任務，立即回傳 `{"status": "started"}`。

## 13. 常見問題

### Q：查詢後沒有資料顯示？
- 確認篩選條件是否過於嚴格，可點擊「**清除**」按鈕移除所有條件後重新查詢。
- 確認資料庫是否已完成初始匯入（查看「最後更新時間」是否有顯示）。

### Q：欄位設定重置了怎麼辦？
- 欄位設定儲存於瀏覽器 `localStorage`，清除瀏覽器快取或使用無痕模式會導致設定消失。
- 可點擊「**欄位設定 → 恢復預設**」快速回到標準配置。

### Q：如何取得最新資料？
- 系統會在每月 5、15、25 日自動更新；若需立即更新，請呼叫 `POST /api/import/trigger`。

### Q：深色模式設定消失？
- 主題設定同樣儲存於 `localStorage`，清除瀏覽器資料會重置為預設淺色模式。

### Q：Email 通知未收到？
- 確認 `.env` 中的 Gmail 設定是否正確，且已啟用「應用程式密碼」。
- 確認 `NOTIFY_EMAIL` 設定的收件地址無誤，並檢查垃圾信件匣。
