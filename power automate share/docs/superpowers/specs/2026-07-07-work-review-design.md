# 工作覆盤（Work Review）功能 設計文件

## 1. 產品定位

在現有「番茄鐘 + 任務管理 + 會議管理」的 Focus Flow 之外，新增一個**覆盤**功能：讓使用者針對某一天（不一定是今天，也可能隔天才補）回顧當天的工作資料（完成的任務、投入的番茄鐘時間、推進中的任務、當天會議），並搭配 3 題主觀反思，最後產出一份固定格式的 Markdown，方便複製或下載到外部系統（例如使用者自己的 journals 專案）留存。

本次不做：

- 覆盤紀錄不落地儲存在 Focus Flow 內（不做覆盤歷史頁面），純粹即時產生、匯出即用
- 不做覆盤提醒、排程或「待覆盤日期清單」
- 不影響首頁摘要列（summary-strip）與熱力圖

## 2. 資料模型異動

### 新增：`PomodoroSession`

目前番茄鐘完成時只會遞增 `task.pomodoros`（全時期累加計數器）與 `state.stats`（全站每日完成數），沒有「某一天、某個任務」層級的紀錄，因此無法回溯過去日期的任務投入時間。新增逐筆紀錄：

| 欄位 | 說明 |
|---|---|
| `id` | 唯一識別碼 |
| `taskId` | 對應完成當下的 `currentTaskId`（可為 `null`，代表當時沒有指定任務） |
| `date` | 完成當下的日期（`YYYY-MM-DD`） |
| `completedAt` | 完成時間戳 |

`completePomodoro()` 在原本遞增 `task.pomodoros` 與呼叫 `incrementTodayStat()` 之外，新增 push 一筆 `PomodoroSession`。既有的 `task.pomodoros` 計數器與 `state.stats` 維持不變，供既有畫面（任務卡片番茄鐘數、熱力圖）沿用，避免動到既有顯示邏輯。

### `Task`：新增兩個欄位

| 欄位 | 說明 |
|---|---|
| `priority` | 字串，如 `P1`–`P4`，可為 `null`（未提供） |
| `status` | 字串，如 `Active`、`未標示`，可為 `null`（未提供） |

手動新增的任務沿用現有欄位，`priority`/`status` 預設為 `null`；只有透過批次匯入且來源文字有提供對應欄位時才會寫入值。

### 批次匯入解析規則擴充

現有規則（會議管理設計文件）在任務區塊只擷取「任務名稱：」，其餘欄位一律捨棄。使用者實際會用到至少兩種 ChatGPT 歸納範本，欄位不完全相同：

- Outlook / Calendar 行動項範本：`任務名稱`／`任務類型`／`來源`／`截止時間`／`是否需要回覆`／`是否跨組織`／`狀態`（無 `Priority`）
- 專案管理型任務範本：欄位不同，包含 `Priority`

因此解析規則改為：在每筆任務區塊（`[數字#]` 到下一個 `[數字#]` 或區塊結尾之間）逐行掃描，除了原本必要的「任務名稱：」，額外找「狀態：」與「Priority：」兩個 key（大小寫不拘），找到就取冒號後文字存入 `Task.status` / `Task.priority`；找不到則該欄位維持 `null`，不視為解析錯誤。其餘欄位（任務類型、來源、截止時間、是否需要回覆、是否跨組織）依然全部捨棄，不落地儲存。

若同一批次文字重複匯入相同任務，比照既有規則不做去重檢查。

## 3. 進入點與頁面結構

- 在全域頁首（不屬於任何分頁內容，三個分頁——工作台／專案管理／熱力圖——都看得到）新增一個「覆盤」按鈕。
- 點擊後切換到獨立的覆盤頁面（非 modal，取代目前分頁內容區域的顯示），頁面內有明確的返回按鈕，回到點擊前所在的分頁。
- 覆盤頁面不屬於 `tab-bar` 的分頁項目，是與三個分頁平行的第四種畫面狀態。

## 4. 覆盤流程

### Step 1：選擇日期

- 一個日期選擇器（`<input type="date">`），無預設值，使用者需自行選擇要覆盤的日期。
- 選定後才顯示後續內容。

### Step 2：資料摘要（唯讀顯示，非問答）

三個區塊，依序：

#### 今天完成的事

- 篩選條件：`Task.completed === true` 且 `Task.completedAt` 落在選定日期
- 每筆顯示：任務標題、當天投入小時（取所有 `PomodoroSession.taskId === 該任務` 且 `date === 選定日期` 的筆數 × 單顆番茄鐘時長，轉換為小時，取一位小數）
- 空狀態文案：「這天沒有完成的任務。」

#### 今天推進中的事

- 篩選條件：`Task.completed === false`（不限建立或到期日期，反映「目前所有還沒做完的事」）
- 每筆顯示：任務標題、狀態（`Task.status`，缺值顯示「未標示」）、Priority（`Task.priority`，缺值顯示「未標示」）
- 空狀態文案：「目前沒有進行中的任務。」

#### 今天的會議

- 篩選條件：`Meeting.date === 選定日期`
- 依 `startTime` 由早到晚排序
- 每筆顯示：開始時間–結束時間（`startTime` + `durationMinutes` 換算）、會議標題
- 空狀態文案：「這天沒有會議紀錄。」

（原構想中的「今天投入時間」彙總區塊已移除，因為內容與「今天完成的事」重複，改為不重複呈現。）

### Step 3：逐題反思問答

依序呈現 3 題，每題可跳過（跳過時最終輸出該段落顯示「今日無資料。」）：

1. 做得好的地方？
2. 卡住或需要調整的地方？
3. 明天的一個小改進？

### Step 4：明天要做的事（可編輯清單）

- 預先帶入 Step 2「今天推進中的事」清單的任務標題，作為草稿
- 使用者可以：刪除某一項（不會影響原本的 `Task` 資料，只影響這份覆盤輸出）、新增自由文字項目、編輯既有項目文字
- 這份清單只用於輸出，不寫回 `Task` 資料

### Step 5：產出結果

依照下方「輸出格式」組成完整 Markdown，畫面上提供：

- 「複製 Markdown」：寫入剪貼簿
- 「下載」：以 `{選定日期}-工作回顧.md` 為檔名觸發瀏覽器下載

不在 Focus Flow 內保存這份輸出；離開覆盤頁面等同捨棄（除非使用者自己複製/下載走）。

## 5. 輸出格式

```markdown
# {YYYY-MM-DD} 工作回顧

## 今天完成的事
- {任務標題}。投入 {X.X} 小時。
（若當天無完成任務：這天沒有完成的任務。）

## 今天推進中的事
- {任務標題}。狀態：{狀態或「未標示」}。Priority：{Priority 或「未標示」}。
（若無進行中任務：目前沒有進行中的任務。）

## 今天的會議
- {開始時間}-{結束時間} {會議標題}。
（若無會議：這天沒有會議紀錄。）

## 復盤筆記
### 做得好的地方
{回答內容或「今日無資料。」}

### 卡住或需要調整的地方
{回答內容或「今日無資料。」}

### 明天的一個小改進
{回答內容或「今日無資料。」}

## 明天要做的事
- [ ] {項目1}
- [ ] {項目2}
```

## 6. 邊界情況

- 選定日期完全沒有任務／會議／番茄鐘資料：三個資料摘要區塊各自顯示對應空狀態文案，反思問答與明天待辦清單照常可以填寫，不因無資料而跳過整個流程。
- 反思三題全部跳過：輸出的「復盤筆記」三段皆顯示「今日無資料。」
- 「明天要做的事」清單被使用者清空：輸出區塊底下不顯示任何 `- [ ]` 項目（維持空的清單標題）。
- 批次匯入文字裡任務區塊只有部分筆有 `狀態`/`Priority`：沒有提供的筆維持 `null`，覆盤畫面顯示「未標示」，不視為解析錯誤或攔阻匯入。
- 同一個 `PomodoroSession` 紀錄的 `taskId` 為 `null`（完成當下沒有指定任務）：不計入任何任務的投入小時，也不影響「今天完成的事」清單（因為該清單以 `Task.completedAt` 篩選，與番茄鐘紀錄是否有任務無關）。

## 7. 驗收清單

- 頁首任一分頁都能看到「覆盤」按鈕，點擊後進入獨立覆盤頁面，可返回原分頁
- 可以手動選擇任一日期進入覆盤
- 選定日期後，能正確顯示當天完成的任務（含正確的投入小時）、目前所有進行中任務（含狀態/Priority）、當天會議清單
- 番茄鐘完成後會新增一筆 `PomodoroSession`，且能被覆盤功能正確加總
- 批次匯入的任務文字如果包含「狀態：」「Priority：」，能正確寫入 `Task.status` / `Task.priority`；沒有提供則維持未標示，不影響匯入其餘任務
- 三題反思問答可逐題填寫或跳過，跳過會在輸出中顯示「今日無資料。」
- 「明天要做的事」預先帶入進行中任務，且可增刪修改後才產出
- 可以複製 Markdown 到剪貼簿，也可以下載成 `.md` 檔案，檔名包含選定日期
- 覆盤內容不會寫回 Focus Flow 既有的 Task/Meeting/Project 資料（明天待辦清單的增刪不影響原始任務）
- 首頁摘要列與熱力圖不受本功能影響
